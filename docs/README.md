## Repository layout

| Path | Purpose |
|------|---------|
| `config/` | Source list and secrets template |
| `data/insight/` | Published daily AI interpretation HTML (committed archive) |
| `data/source-packs/` | Raw source material for brief generation |
| `data/daily/`, `data/segments/` | Committed digest archive |
| `dist/` | Local build output for GitHub Pages (gitignored, regenerated by CI) |
| `scripts/` | Pipeline entry points |
| `staging/drafts/` | Codex output before publish (gitignored, auto-deleted after publish) |
| `tmp/` | Local scratch files (gitignored, safe to delete anytime) |

Root keeps only project metadata: `README.md`, `requirements.txt`, `pytest.ini`, `CNAME`.

### Brief lifecycle

| Stage | Location | Meaning |
|-------|----------|---------|
| Draft | `staging/drafts/firsthand-ai-brief-*.html` | Codex just generated it; not published yet |
| Published (source of truth) | `data/insight/YYYY-MM-DD-ai-brief.html` | Committed archive in this repo |
| Deployed (main site) | `dist/archive/` | Build output for `ai.prov1dence.top` (gitignored) |
| Deployed (insight site) | `archive/` in `firsthand-ai-insight` | Synced copy for `insight.ai.prov1dence.top` |

`staging/drafts/` is an inbox, not an archive. `publish_ai_brief.py` copies into
`data/insight/` and removes the draft. `dist/` is rebuilt by `archive_data.py` and
can be wiped locally at any time.
